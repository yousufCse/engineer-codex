# Part 4 — নেটওয়ার্ক লেয়ার

> IP Address, Subnetting, Routing, NAT — এই চারটি ছাড়া ইন্টারনেট অচল।

## অধ্যায় ৯: IP Address — IPv4 ও IPv6

### ৯.১ IP Address কী?

IP (Internet Protocol) Address হলো নেটওয়ার্কে প্রতিটি ডিভাইসের লজিক্যাল ঠিকানা। MAC Address যেমন হার্ডওয়্যারের ঠিকানা, IP Address তেমন নেটওয়ার্কের ঠিকানা।

```
ডাকব্যবস্থার সাথে তুলনা:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  MAC Address = আপনার জাতীয় পরিচয়পত্র নম্বর
               (স্থায়ী, সব জায়গায় একই)

  IP Address  = আপনার বর্তমান বাসার ঠিকানা
               (পরিবর্তনযোগ্য, নেটওয়ার্ক অনুযায়ী ভিন্ন)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

### ৯.২ IPv4 — Internet Protocol Version 4

**IPv4** হলো ৩২ বিটের ঠিকানা পদ্ধতি।

```
IPv4 গঠন:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  192    .   168   .    1    .   100
   │         │         │         │
  8 bits   8 bits   8 bits   8 bits
(0-255)  (0-255)  (0-255)  (0-255)
                                    মোট = 32 bits

বাইনারিতে:
  192 = 11000000
  168 = 10101000
    1 = 00000001
  100 = 01100100

  সম্পূর্ণ: 11000000.10101000.00000001.01100100

মোট সম্ভাব্য IPv4 ঠিকানা:
  2³² = 4,294,967,296 ≈ 4.3 বিলিয়ন

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ৯.৩ IPv4 Classes

```
IPv4 Class সিস্টেম:
┌───────┬──────────────┬───────────────┬───────────┬──────────┐
│ Class │ প্রথম Octet  │ Range         │ Default   │ Hosts/   │
│       │              │               │ Subnet    │ Network  │
├───────┼──────────────┼───────────────┼───────────┼──────────┤
│   A   │ 0xxxxxxx     │ 1–126         │ /8        │ 16M      │
│   B   │ 10xxxxxx     │ 128–191       │ /16       │ 65,534   │
│   C   │ 110xxxxx     │ 192–223       │ /24       │ 254      │
│   D   │ 1110xxxx     │ 224–239       │ N/A       │ Multicast│
│   E   │ 1111xxxx     │ 240–255       │ N/A       │ Reserved │
└───────┴──────────────┴───────────────┴───────────┴──────────┘

127.x.x.x = Loopback (localhost) — নেটওয়ার্কে যায় না
```

### ৯.৪ Private vs Public IP

```
Private IP Ranges (RFC 1918):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Class A: 10.0.0.0     –  10.255.255.255   (10/8)
  Class B: 172.16.0.0   –  172.31.255.255   (172.16/12)
  Class C: 192.168.0.0  –  192.168.255.255  (192.168/16)

এই IP গুলো ইন্টারনেটে route করা হয় না।
শুধু Private Network-এ ব্যবহার করা যায়।
NAT দিয়ে Public IP-তে রূপান্তর করতে হয়।

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

বিশেষ IP ঠিকানা:
  127.0.0.1        → Loopback (localhost)
  0.0.0.0          → এই নেটওয়ার্ক (unspecified)
  255.255.255.255  → Limited Broadcast
  169.254.0.0/16   → APIPA (DHCP ব্যর্থ হলে auto-assign)
  224.0.0.0/4      → Multicast
```

### ৯.৫ IPv6 — ভবিষ্যতের ঠিকানা

IPv4-এর 4.3 বিলিয়ন ঠিকানা শেষ হয়ে আসছে। IPv6 এনেছে **128 বিটের** ঠিকানা।

```
IPv6 গঠন:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  2001:0db8:85a3:0000:0000:8a2e:0370:7334
    │    │    │    │    │    │    │    │
   16   16   16   16   16   16   16   16  bits (প্রতিটি group)
   মোট = 128 bits

  সংক্ষিপ্ত লেখার নিয়ম:
  • শূন্য group → ::
  • Leading zeros বাদ দেওয়া
  2001:0db8:0000:0000:0000:0000:0000:0001
  = 2001:db8::1  (সংক্ষিপ্ত)

মোট সম্ভাব্য ঠিকানা:
  2¹²⁸ ≈ 340 undecillion (340 × 10³⁶)
  = পৃথিবীর প্রতিটি বালুকণার চেয়ে বেশি!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**IPv4 vs IPv6 তুলনা:**

| বৈশিষ্ট্য | IPv4 | IPv6 |
|-----------|------|------|
| দৈর্ঘ্য | 32 bit | 128 bit |
| ঠিকানা সংখ্যা | ~4.3 বিলিয়ন | ~340 undecillion |
| Notation | Decimal (192.168.1.1) | Hex (2001:db8::1) |
| Header | 20 bytes | 40 bytes (fixed) |
| NAT | প্রয়োজন | প্রয়োজন নেই |
| IPsec | Optional | Built-in |
| Broadcast | আছে | নেই (Multicast দিয়ে) |

**বিশেষ IPv6 ঠিকানা:**
```
::1              → Loopback (IPv4-এর 127.0.0.1)
fe80::/10        → Link-local (LAN-এ auto-assign)
fc00::/7         → Unique-local (Private, IPv4-এর RFC 1918)
ff00::/8         → Multicast
2001:db8::/32    → Documentation (উদাহরণে ব্যবহার)
```

### ৯.৬ Unicast, Multicast, Anycast, Broadcast

```
ডেটা পাঠানোর ধরন:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Unicast: একজনকে পাঠাও
  ┌────┐         ┌────┐
  │ A  │────────▶│ B  │   শুধু B পাবে
  └────┘         └────┘

  Broadcast: সবাইকে পাঠাও
  ┌────┐    ┌────┐┌────┐┌────┐
  │ A  │───▶│ B  ││ C  ││ D  │   সবাই পাবে
  └────┘    └────┘└────┘└────┘

  Multicast: নির্দিষ্ট দলকে পাঠাও
  ┌────┐    ┌────┐
  │ A  │───▶│ B  │   B ও D subscribe করেছে
  └────┘  ↗ └────┘   শুধু তারা পাবে
          │ ┌────┐
          └▶│ D  │
            └────┘

  Anycast: (IPv6-এ) কাছের একজনকে পাঠাও
  ┌────┐    ┌──────────┐ ← London
  │ A  │───▶│ Same IP! │ ← Singapore
  └────┘    └──────────┘ ← Mumbai  (কাছেরটা পাবে)
  (CDN, DNS এ ব্যবহৃত)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## অধ্যায় ১০: Subnetting ও CIDR

### ১০.১ Subnet কেন দরকার?

একটি বড় নেটওয়ার্ককে ছোট ছোট অংশে ভাগ করাই **Subnetting**।

```
কেন Subnetting?

  ❌ সমস্যা (Subnetting ছাড়া):
  পুরো অফিস একটি নেটওয়ার্কে → একটি Broadcast
  সবার কাছে যায় → নেটওয়ার্ক ধীর হয়

  ✅ সমাধান (Subnetting দিয়ে):
  HR:      192.168.1.0/24   (254 host)
  IT:      192.168.2.0/24   (254 host)
  Finance: 192.168.3.0/24   (254 host)

  প্রতিটি বিভাগের Broadcast নিজেদের মধ্যে সীমাবদ্ধ
```

---

### ১০.২ Subnet Mask

```
Subnet Mask বোঝা:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  IP Address:   192 . 168 .  1  . 100
  Subnet Mask:  255 . 255 . 255.   0
                └────Network────┘ └Host┘

  Subnet Mask = Network অংশকে 1 দিয়ে চিহ্নিত করে
  বাইনারিতে:
    255.255.255.0 = 11111111.11111111.11111111.00000000
                    └────── 24 ones ──────────┘└─8 zeros─┘

  এই IP-এর:
    Network: 192.168.1.0
    Host range: 192.168.1.1 – 192.168.1.254
    Broadcast: 192.168.1.255

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

### ১০.৩ CIDR Notation

**CIDR (Classless Inter-Domain Routing)** — Subnet Mask সংক্ষেপে লেখার পদ্ধতি।

```
CIDR = IP/Prefix Length

  192.168.1.0/24
               ↑
               24টি 1 (Network bits)
               বাকি 32-24 = 8 bits Host-এর জন্য
               2⁸ - 2 = 254টি host সম্ভব

CIDR থেকে সব বের করা:

  192.168.1.0/24:
    Subnet Mask:  255.255.255.0
    Network:      192.168.1.0
    First Host:   192.168.1.1
    Last Host:    192.168.1.254
    Broadcast:    192.168.1.255
    Total Hosts:  2⁸ - 2 = 254

  10.0.0.0/8:
    Subnet Mask:  255.0.0.0
    Total Hosts:  2²⁴ - 2 = 16,777,214

  192.168.1.128/26:
    Subnet Mask:  255.255.255.192
    Network:      192.168.1.128
    First Host:   192.168.1.129
    Last Host:    192.168.1.190
    Broadcast:    192.168.1.191
    Total Hosts:  2⁶ - 2 = 62
```

---

### ১০.৪ Subnetting সম্পূর্ণ উদাহরণ

**সমস্যা:** 192.168.1.0/24 নেটওয়ার্ককে ৪টি সমান subnet-এ ভাগ করুন।

```
সমাধান:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

৪টি subnet → 2² = 4, তাই ২টি অতিরিক্ত bit দরকার
নতুন prefix = /24 + 2 = /26

Subnet 1: 192.168.1.0/26
  Network:   192.168.1.0
  Hosts:     192.168.1.1 – 192.168.1.62
  Broadcast: 192.168.1.63

Subnet 2: 192.168.1.64/26
  Network:   192.168.1.64
  Hosts:     192.168.1.65 – 192.168.1.126
  Broadcast: 192.168.1.127

Subnet 3: 192.168.1.128/26
  Network:   192.168.1.128
  Hosts:     192.168.1.129 – 192.168.1.190
  Broadcast: 192.168.1.191

Subnet 4: 192.168.1.192/26
  Network:   192.168.1.192
  Hosts:     192.168.1.193 – 192.168.1.254
  Broadcast: 192.168.1.255

প্রতিটি subnet-এ 62টি host!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## অধ্যায় ১১: Routing — ডেটা প্যাকেটের পথ নির্ধারণ

### ১১.১ Routing কী?

Routing হলো একটি নেটওয়ার্ক থেকে আরেকটি নেটওয়ার্কে প্যাকেট পাঠানোর প্রক্রিয়া। Router এই কাজ করে।

```
Router-এর কাজ:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────────────────────────────┐
  │                    Router                        │
  │                                                  │
  │  Routing Table:                                  │
  │  ┌────────────────┬──────────┬─────────────────┐ │
  │  │ Destination    │ Next Hop │ Interface       │ │
  │  ├────────────────┼──────────┼─────────────────┤ │
  │  │ 192.168.1.0/24 │ directly │ eth0            │ │
  │  │ 10.0.0.0/8     │ 203.1.1.1│ eth1            │ │
  │  │ 0.0.0.0/0      │ 1.2.3.4  │ eth1 (default) │ │
  │  └────────────────┴──────────┴─────────────────┘ │
  │                                                  │
  │  প্রতিটি প্যাকেটের জন্য:                          │
  │  1. Destination IP দেখো                          │
  │  2. Routing Table-এ সবচেয়ে specific match খোঁজো │
  │  3. সেই next hop-এ পাঠাও                         │
  └──────────────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ১১.২ Longest Prefix Match

```
Longest Prefix Match:

Routing Table:
  10.0.0.0/8     → Interface A
  10.1.0.0/16    → Interface B
  10.1.1.0/24    → Interface C
  0.0.0.0/0      → Interface D (default)

Destination: 10.1.1.50

Matches:
  ✓ 10.0.0.0/8     (8 bits match)
  ✓ 10.1.0.0/16    (16 bits match)
  ✓ 10.1.1.0/24    (24 bits match) ← সবচেয়ে specific!
  ✓ 0.0.0.0/0      (0 bits match)

সিদ্ধান্ত: Interface C-তে পাঠাও (longest = /24)
```

### ১১.৩ Static vs Dynamic Routing

```
Static Routing:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Administrator manually entry দেয়:
  ip route 192.168.2.0/24 via 10.0.0.1

  ✅ সুবিধা: সহজ, predictable, কম CPU
  ❌ অসুবিধা: manual update দরকার, scalable নয়
  📍 ব্যবহার: ছোট নেটওয়ার্ক, specific route

Dynamic Routing:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Router নিজে নিজে route শেখে ও আপডেট করে

  ✅ সুবিধা: automatic, scalable, self-healing
  ❌ অসুবিধা: বেশি CPU/memory, জটিল
  📍 ব্যবহার: বড় নেটওয়ার্ক, ISP

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ১১.৪ Routing Protocols

```
Routing Protocol তুলনা:
┌──────────┬──────────────┬───────────────┬─────────────────┐
│ Protocol │ ধরন          │ Algorithm     │ ব্যবহার         │
├──────────┼──────────────┼───────────────┼─────────────────┤
│ RIP      │ Distance     │ Bellman-Ford  │ ছোট নেটওয়ার্ক │
│          │ Vector       │ Hop count     │ (obsolete)      │
├──────────┼──────────────┼───────────────┼─────────────────┤
│ OSPF     │ Link State   │ Dijkstra      │ Enterprise      │
│          │              │               │ নেটওয়ার্ক       │
├──────────┼──────────────┼───────────────┼─────────────────┤
│ BGP      │ Path Vector  │ Best Path     │ ইন্টারনেট ISP  │
│          │              │ Selection     │ (ইন্টারনেটের   │
│          │              │               │ মেরুদণ্ড)      │
└──────────┴──────────────┴───────────────┴─────────────────┘
```

**BGP (Border Gateway Protocol):**
```
BGP = ইন্টারনেটের গ্লু

  ISP-A ←──BGP──→ ISP-B ←──BGP──→ ISP-C
  (AS 1234)        (AS 5678)        (AS 9012)

  AS = Autonomous System (একটি নেটওয়ার্কের এলাকা)

  বাংলাদেশ → BTCL (AS 17501) → international

2021 Facebook outage:
  Facebook ভুল BGP update দিলো →
  তাদের IP prefix ইন্টারনেট থেকে হারিয়ে গেলো →
  6 ঘণ্টা সম্পূর্ণ বন্ধ!
```

## অধ্যায় ১২: NAT ও PAT — একটি IP দিয়ে অনেক ডিভাইস

### ১২.১ NAT কেন দরকার?

IPv4 ঠিকানা মাত্র 4.3 বিলিয়ন। কিন্তু বিশ্বে ডিভাইস আরও বেশি। **NAT (Network Address Translation)** এই সমস্যার সমাধান।

```
NAT-এর মূল ধারণা:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  বাড়ির ১০০টি ডিভাইস → ১টি Public IP দিয়ে ইন্টারনেট ব্যবহার

  Private Network          Router/NAT         Internet
  ┌──────────────┐        ┌─────────┐
  │ 192.168.1.2  │───────▶│         │────▶ 203.1.1.1 (Public)
  │ 192.168.1.3  │───────▶│  NAT    │
  │ 192.168.1.4  │───────▶│         │
  │ ...          │        └─────────┘
  └──────────────┘

  ইন্টারনেট শুধু 203.1.1.1 দেখে — ভেতরের কিছু জানে না।

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

### ১২.২ NAT এর প্রকারভেদ

```
তিনটি NAT:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

① Static NAT:
  একটি Private IP → সবসময় একটি নির্দিষ্ট Public IP
  192.168.1.10 → 203.1.1.10 (সবসময়)
  ব্যবহার: Web server (বাইরে থেকে access দরকার)

② Dynamic NAT:
  একটি Pool থেকে IP বরাদ্দ (first-come-first-serve)
  192.168.1.10 → 203.1.1.5 (যেটা ফাঁকা সেটা)
  Pool শেষ হলে নতুন connection ব্যর্থ হয়

③ PAT / NAT Overload (সবচেয়ে বেশি ব্যবহৃত):
  অনেক Private IP → একটি Public IP (Port দিয়ে আলাদা)
  আপনার বাড়ির Router এটাই করে!

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ১২.৩ PAT (Port Address Translation) বিস্তারিত

```
PAT NAT Table:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌──────────────────┬────────────────────┬──────────────┐
│ Inside Local     │ Inside Global      │ Outside      │
│ (Private)        │ (Public)           │ (Server)     │
├──────────────────┼────────────────────┼──────────────┤
│192.168.1.2:54321 │203.1.1.1:10001     │8.8.8.8:443   │
│192.168.1.3:54322 │203.1.1.1:10002     │8.8.8.8:443   │
│192.168.1.4:49000 │203.1.1.1:10003     │1.1.1.1:53    │
└──────────────────┴────────────────────┴──────────────┘

প্রক্রিয়া:
1. 192.168.1.2:54321 → 8.8.8.8:443 পাঠাতে চায়
2. Router: Source IP+Port পাল্টায়
   192.168.1.2:54321 → 203.1.1.1:10001
3. Table-এ entry রাখে
4. Response আসলে: 203.1.1.1:10001 → 192.168.1.2:54321
   Table দেখে রূপান্তর করে ফেরত পাঠায়

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ১২.৪ NAT-এর সমস্যা ও Flutter-এ প্রভাব

```
NAT-এর সমস্যা:
  ① Peer-to-Peer সংযোগ কঠিন
     দুটো NAT-এর পেছনের device সরাসরি কথা বলতে পারে না
     Flutter-এ: WebRTC video call, P2P file sharing

  ② Server হওয়া কঠিন
     বাইরে থেকে Private IP access করা যায় না
     সমাধান: Port Forwarding

  ③ Performance:
     NAT table maintain করতে CPU লাগে

Port Forwarding:
  Router-এ rule: External port 8080 → 192.168.1.10:80
  এখন বাইরে থেকে 203.1.1.1:8080 → 192.168.1.10:80

Flutter অ্যাপে NAT Traversal:
  STUN server: "তোমার Public IP কী?"
  TURN server: relay server (NAT পার হতে না পারলে)
  ICE: সব পথ চেষ্টা করে সেরাটা বেছে নেয়
  WebRTC = এই সব মিলিয়ে কাজ করে
```
