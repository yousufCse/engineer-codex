# Part 1 — ভিত্তি

> কম্পিউটার নেটওয়ার্কিং বোঝার আগে কম্পিউটার নিজে কিভাবে তথ্য বোঝে সেটা জানতে হবে। এই পার্টে আমরা একেবারে শূন্য থেকে শুরু করব।

---

## অধ্যায় ১: বিট, বাইট ও বাইনারি — সবকিছুর শুরু এখানে

### ১.১ কেন বাইনারি?

আপনার Flutter অ্যাপ যখন একটি API কল করে, তখন JSON ডেটা নেটওয়ার্কে পাঠানো হয়। কিন্তু সেই JSON আসলে তার বর্তমান রূপে পাঠানো হয় না — সবকিছু রূপান্তরিত হয় বিদ্যুতের স্পন্দনে, আলোর ঝলকে, বা রেডিও তরঙ্গে।

কম্পিউটারের মৌলিক উপাদান হলো **ট্রানজিস্টর** — একটি ক্ষুদ্র বৈদ্যুতিক সুইচ। এই সুইচ হয় ON নয়তো OFF। এই দুই অবস্থা থেকে জন্ম নেয় বাইনারি সংখ্যা পদ্ধতি।

```
বাস্তব জগতের সাদৃশ্য:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  💡 বাল্ব জ্বলছে  →  1  →  HIGH Voltage (~5V)
  💡 বাল্ব নেভা   →  0  →  LOW Voltage  (~0V)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

### ১.২ বিট (Bit) — তথ্যের ক্ষুদ্রতম একক

**Bit = Binary Digit**। এটি শুধু দুটো মান ধারণ করতে পারে: **0** অথবা **1**।

```
একটি Bit:
  ┌───┐        ┌───┐
  │ 0 │  অথবা  │ 1 │
  └───┘        └───┘
  (OFF)         (ON)
```

**Flutter উদাহরণ:** আপনার অ্যাপের `bool isLoggedIn = true;` — এটি মেমরিতে একটি বিট হিসেবে সংরক্ষিত হতে পারে।

---

### ১.৩ বাইট (Byte) — অর্থবহ একক

**৮টি বিট = ১টি বাইট**। ৮ বিট দিয়ে **2⁸ = 256** টি আলাদা মান প্রকাশ করা যায়।

```
১ Byte = 8 Bits:
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 0 │ 1 │ 0 │ 0 │ 0 │ 0 │ 0 │ 1 │  = 65 = 'A' (ASCII)
└───┴───┴───┴───┴───┴───┴───┴───┘
  ↑                           ↑
Most Significant Bit (MSB)  Least Significant Bit (LSB)
```

---

### ১.৪ বাইনারি থেকে দশমিক রূপান্তর

দশমিকে প্রতিটি ঘরের মান ১০-এর ঘাত। বাইনারিতে প্রতিটি ঘরের মান ২-এর ঘাত।

```
বাইনারি 1000 0111 → দশমিক:

  বিট:   1    0    0    0    0    1    1    1
  ঘাত:  2⁷   2⁶   2⁵   2⁴   2³   2²   2¹   2⁰
  মান: 128   64   32   16    8    4    2    1

  গণনা: (1×128) + (0×64) + (0×32) + (0×16)
        + (0×8) + (1×4) + (1×2) + (1×1)
      = 128 + 0 + 0 + 0 + 0 + 4 + 2 + 1
      = 135 ✓
```

### ১.৫ দশমিক থেকে বাইনারি রূপান্তর

**পদ্ধতি:** ২ দিয়ে ভাগ করতে থাকুন, ভাগশেষগুলো নিচ থেকে উপরে পড়ুন।

```
দশমিক 135 → বাইনারি:

  135 ÷ 2 = 67  ভাগশেষ 1  ←── শেষ বিট (LSB)
   67 ÷ 2 = 33  ভাগশেষ 1
   33 ÷ 2 = 16  ভাগশেষ 1
   16 ÷ 2 =  8  ভাগশেষ 0
    8 ÷ 2 =  4  ভাগশেষ 0
    4 ÷ 2 =  2  ভাগশেষ 0
    2 ÷ 2 =  1  ভাগশেষ 0
    1 ÷ 2 =  0  ভাগশেষ 1  ←── প্রথম বিট (MSB)

উপরে থেকে নিচে পড়লে: 10000111 ✓
```

### ১.৬ হেক্সাডেসিমাল (Hexadecimal) — নেটওয়ার্কিংয়ের ভাষা

বাইনারি দীর্ঘ হয়ে যায়, তাই নেটওয়ার্কিংয়ে **হেক্স (Base-16)** ব্যাপকভাবে ব্যবহৃত হয়। হেক্সে ০-৯ এবং A-F (মোট ১৬টি অঙ্ক)।

```
হেক্স টেবিল:
┌──────────┬─────────┬─────────────┐
│ দশমিক   │ বাইনারি │ হেক্স       │
├──────────┼─────────┼─────────────┤
│   0      │  0000   │     0       │
│   1      │  0001   │     1       │
│   9      │  1001   │     9       │
│  10      │  1010   │     A       │
│  11      │  1011   │     B       │
│  12      │  1100   │     C       │
│  13      │  1101   │     D       │
│  14      │  1110   │     E       │
│  15      │  1111   │     F       │
│  16      │  0001 0000  │    10   │
│ 255      │  1111 1111  │    FF   │
└──────────┴─────────┴─────────────┘

৪টি বিট = ১টি হেক্স ডিজিট
৮টি বিট (১ বাইট) = ২টি হেক্স ডিজিট
```

**নেটওয়ার্কে হেক্স কোথায় দেখবেন:**
- MAC Address: `2C:54:91:88:C9:E3` ← প্রতিটি জোড়া = ১ বাইট
- IPv6: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
- TLS Certificate thumbprint
- HTTP Header values

```dart
// Flutter-এ হেক্স ব্যবহার:
// Color হেক্সে: #FF5722
Color myColor = Color(0xFFFF5722);
//              ├──┤├──┤├──┤├──┤
//             Alpha  R   G   B
//             (FF=255, FF=255, 57=87, 22=34)
```

### ১.৭ ডেটার একক — KB থেকে TB পর্যন্ত

```
ডেটা এককের সিঁড়ি:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1 Bit          = 0 অথবা 1
  1 Byte         = 8 Bits
  ────────────────────────────────────────────────────
  1 Kilobyte  (KB) = 1,024 Bytes       ≈ 1 হাজার বাইট
  1 Megabyte  (MB) = 1,024 KB          ≈ 10 লক্ষ বাইট
  1 Gigabyte  (GB) = 1,024 MB          ≈ 100 কোটি বাইট
  1 Terabyte  (TB) = 1,024 GB          ≈ 1 লক্ষ কোটি বাইট
  1 Petabyte  (PB) = 1,024 TB
  1 Exabyte   (EB) = 1,024 PB
  ────────────────────────────────────────────────────
  ⚠️  নেটওয়ার্ক Speed সাধারণত bits/sec-এ পরিমাপ হয়!
      1 Mbps ≠ 1 MB/s
      1 Mbps = 1,000,000 bits/sec = 125,000 Bytes/sec ≈ 0.125 MB/s

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**বাস্তব উদাহরণ:**

| ফাইল/ডেটা | আনুমানিক আকার |
|-----------|--------------|
| একটি ইংরেজি অক্ষর | 1 Byte |
| একটি বাংলা অক্ষর (UTF-8) | 3 Bytes |
| একটি ছোট JSON response | 1-5 KB |
| একটি HD ছবি (JPEG) | 100 KB – 5 MB |
| একটি Flutter APK | 10-30 MB |
| একটি HD মুভি | 1-4 GB |
| Google-এর দৈনিক ডেটা প্রসেসিং | ~100 PB |

### ১.৮ Character Encoding — বর্ণ কিভাবে বাইনারিতে রূপান্তরিত হয়

```
ASCII (American Standard Code for Information Interchange):
'A' = 65  = 01000001
'B' = 66  = 01000010
'a' = 97  = 01100001
'1' = 49  = 00110001
' ' = 32  = 00100000

UTF-8 (Universal, বাংলা সহ সব ভাষা):
'A'   = 1 Byte  = 01000001
'©'   = 2 Bytes = 11000010 10101001
'অ'  = 3 Bytes = 11100000 10101110 10000000
'😊' = 4 Bytes = 11110000 10011111 10011000 10001010
```

**Flutter-এ প্রাসঙ্গিকতা:**

```dart
// Flutter-এ String সবসময় UTF-16 (Dart language)
String bangla = 'নেটওয়ার্ক';
List<int> utf8Bytes = utf8.encode(bangla);
print(utf8Bytes.length); // প্রতিটি বাংলা অক্ষর = 3 bytes
// তাই 'নেটওয়ার্ক' = ১০ অক্ষর × ৩ = ৩০ bytes

// Network-এ পাঠানোর সময় সবসময় encoding specify করুন
// Content-Type: application/json; charset=utf-8
```

### ১.৯ বিটস ও নেটওয়ার্ক স্পিড

নেটওয়ার্কের গতি মাপা হয় **bits per second (bps)** এ:

```
স্পিড তুলনা:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  56 Kbps    →  পুরনো ডায়াল-আপ মডেম
  1 Mbps     →  পুরনো DSL সংযোগ
  10 Mbps    →  সাধারণ ব্রডব্যান্ড
  100 Mbps   →  দ্রুত ব্রডব্যান্ড / Fast Ethernet
  1 Gbps     →  Gigabit Ethernet / আধুনিক Wi-Fi
  10 Gbps    →  ডেটাসেন্টারের সার্ভার সংযোগ
  100 Gbps   →  ইন্টারনেটের মেরুদণ্ড (Backbone)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

গণনা উদাহরণ:
  আপনার ইন্টারনেট স্পিড 100 Mbps।
  একটি 50 MB ফাইল ডাউনলোড করতে কতক্ষণ?

  50 MB = 50 × 8 Mb = 400 Mb (bits-এ রূপান্তর)
  সময় = 400 Mb ÷ 100 Mbps = 4 সেকেন্ড
```

## অধ্যায় ২: ডেটা কমিউনিকেশন — তথ্য কিভাবে চলাচল করে

> *"দুটো কম্পিউটারের মধ্যে যোগাযোগ মানুষের কথোপকথনের মতোই — একজন বলে, অন্যজন শোনে, তারপর উত্তর দেয়।"*

### ২.১ কমিউনিকেশন কী?

ডেটা কমিউনিকেশন মানে এক জায়গা থেকে আরেক জায়গায় তথ্য পাঠানো। এর জন্য দরকার:

```
ডেটা কমিউনিকেশনের পাঁচটি মূল উপাদান:

  ┌────────────┐      ┌─────────────┐      ┌────────────┐
  │  Sender    │─────▶│   Medium    │─────▶│  Receiver  │
  │ (প্রেরক)  │      │  (মাধ্যম)   │      │ (গ্রহীতা)  │
  └────────────┘      └─────────────┘      └────────────┘
        │                    │                    │
   Message তৈরি        সিগন্যাল বহন          Message গ্রহণ
   Encoding করে         করে নেয়             Decoding করে

  + Protocol (নিয়মকানুন)
  + Message (যে তথ্য পাঠানো হচ্ছে)
```

**বাস্তব উদাহরণ:**
- আপনার Flutter অ্যাপ (Sender) → ইন্টারনেট (Medium) → API Server (Receiver)
- আপনার মুখ (Sender) → বাতাস (Medium) → বন্ধুর কান (Receiver)

### ২.২ সিগন্যাল — তথ্য বহনের বাহন

সিগন্যাল হলো তথ্য বহনকারী শক্তি। দুই ধরনের সিগন্যাল:

#### Analog Signal (অ্যানালগ সিগন্যাল)

```
Analog Signal — ক্রমাগত পরিবর্তনশীল তরঙ্গ:

  ↑ Amplitude
  │    ╭──╮         ╭──╮
  │  ╭╯  ╰╮       ╭╯  ╰╮
  │──╯    ╰╮     ╭╯    ╰╮──▶ সময়
  │        ╰─────╯      ╰─
  │
  উদাহরণ: ফোন কলের ভয়েস, রেডিও তরঙ্গ
```

#### Digital Signal (ডিজিটাল সিগন্যাল)

```
Digital Signal — শুধু HIGH বা LOW:

  ↑ Voltage
  │ ┌──┐     ┌────┐  ┌─┐
  │ │  │     │    │  │ │
  │─┘  └─────┘    └──┘ └───▶ সময়
  │
  মান: 1 0   1  1  1  1 0
  উদাহরণ: Ethernet, USB, HDMI
```

| বৈশিষ্ট্য | Analog | Digital |
|-----------|--------|---------|
| মান | যেকোনো মান | শুধু 0 বা 1 |
| Noise-এ প্রভাব | বেশি প্রভাবিত | কম প্রভাবিত |
| দূরত্ব | কম | বেশি |
| ব্যবহার | পুরনো ফোন, রেডিও | ইন্টারনেট, কম্পিউটার |

---

### ২.৩ Bandwidth — পাইপের মোটা হওয়া

**Bandwidth** হলো প্রতি সেকেন্ডে কতটুকু ডেটা পাঠানো সম্ভব তার সর্বোচ্চ সীমা।

```
পানির পাইপের সাথে তুলনা:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  সরু পাইপ (কম Bandwidth):
  ══════▶ [====] ══════▶   10 Mbps
          ডেটা

  মোটা পাইপ (বেশি Bandwidth):
  ══════▶ [══════════] ══════▶   1 Gbps
               ডেটা

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Bandwidth বেশি = একসাথে বেশি ডেটা যেতে পারে
```

**গুরুত্বপূর্ণ:** Bandwidth হলো সর্বোচ্চ ধারণক্ষমতা, কিন্তু **Throughput** হলো আসল পাওয়া গতি।

---

### ২.৪ Latency — দেরির বিজ্ঞান

**Latency (বিলম্ব)** হলো ডেটা প্রেরক থেকে গ্রহীতায় পৌঁছাতে কত সময় লাগে।

```
Latency এর উপাদান:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Total Latency = Propagation + Transmission + Processing + Queuing

  ① Propagation Delay: সিগন্যাল মাধ্যমে ভ্রমণের সময়
     (আলোর গতি ≈ 300,000 km/s)
     ঢাকা → লন্ডন (7,500 km) = 7,500/300,000 × 1000 = 25 ms

  ② Transmission Delay: ডেটা পাঠানোর সময়
     1 MB ফাইল ÷ 100 Mbps = 80 ms

  ③ Processing Delay: রাউটার/সুইচ ডেটা প্রসেস করার সময়
     সাধারণত 1-10 ms

  ④ Queuing Delay: নেটওয়ার্ক জ্যামে অপেক্ষার সময়
     পরিবর্তনশীল (0 – কয়েক সেকেন্ড)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**RTT (Round Trip Time):** একটি প্যাকেট যাওয়া এবং আসার মোট সময়। `ping` কমান্ড RTT দেখায়।

```
বাস্তব Latency মানদণ্ড:
  < 1 ms    → স্থানীয় নেটওয়ার্ক (LAN)
  1–10 ms   → একই দেশের সার্ভার
  10–100 ms → প্রতিবেশী দেশের সার্ভার
  100–300ms → আন্তঃমহাদেশীয় সংযোগ
  > 300 ms  → অনুপযুক্ত (ব্যবহারকারী অস্বস্তি অনুভব করে)
```

**Flutter App-এ প্রভাব:**
- API response > 200ms → ব্যবহারকারী অস্বস্তি বোধ করেন
- WebSocket latency > 100ms → লাইভ চ্যাট জড়িয়ে যায়
- ভিডিও স্ট্রিম latency > 150ms → কথোপকথনে সমস্যা

---

### ২.৫ Throughput vs Bandwidth — বাস্তবতার পার্থক্য

```
উদাহরণ: ঢাকা–চট্টগ্রাম হাইওয়ে

  Bandwidth   = হাইওয়েতে মোট লেনের সংখ্যা
  Throughput  = আসলে কতগুলো গাড়ি প্রতি ঘণ্টায় যাচ্ছে
  Latency     = একটি গাড়ির ঢাকা থেকে চট্টগ্রাম পৌঁছাতে সময়

  রাশ আওয়ারে হাইওয়ে 4 লেনের (Bandwidth = 4)
  কিন্তু জ্যামের কারণে মাত্র 1 লেন পরিমাণ গাড়ি যায় (Throughput = 1)
```

```
কেন Throughput < Bandwidth?
  • Network Congestion (জ্যাম)
  • Protocol Overhead (TCP হেডার, ইত্যাদি)
  • Packet Loss ও Retransmission
  • Full-duplex হলে প্রতিটি দিকে অর্ধেক করে
  • Signal interference বা noise
```

---

### ২.৬ Jitter — অনিশ্চিত দেরি

**Jitter** হলো প্যাকেটের আসার সময়ের অনিয়মিততা।

```
Jitter বোঝার উদাহরণ:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  প্রেরক প্রতি ১০ ms-এ প্যাকেট পাঠায়:
  পাঠানো:  [P1]──10ms──[P2]──10ms──[P3]──10ms──[P4]

  Jitter ছাড়া (আদর্শ):
  পাওয়া:  [P1]──10ms──[P2]──10ms──[P3]──10ms──[P4]

  Jitter সহ (বাস্তব):
  পাওয়া:  [P1]──8ms──[P2]────15ms────[P3]──7ms──[P4]
                              ↑
                        এই অনিয়মিততাই Jitter

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**Jitter কোথায় সমস্যা করে:**
- VoIP (ভয়েস কল) — কথা কেটে যায়
- ভিডিও কনফারেন্স — ছবি জমে যায়
- অনলাইন গেম — lag spike
- Live streaming — buffering

**সমাধান:** Jitter Buffer — প্যাকেটগুলো একটু ধরে রেখে নিয়মিত বিরতিতে ছাড়া হয়।

### ২.৭ Transmission Modes — কথা বলার পদ্ধতি

```
তিনটি মোড:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

① Simplex (এক দিক):
   A ────────────────────▶ B
   শুধু A পাঠাতে পারে, B শুধু গ্রহণ করে
   উদাহরণ: TV broadcast, রেডিও, keyboard→CPU

② Half-Duplex (পালাক্রমে):
   A ────────────────────▶ B   (A পাঠানোর সময়)
   A ◀──────────────────── B   (B পাঠানোর সময়)
   একসাথে দুজন পাঠাতে পারে না
   উদাহরণ: Walkie-talkie, পুরনো Wi-Fi (802.11a/b/g)

③ Full-Duplex (একসাথে দুইদিক):
   A ────────────────────▶ B
   A ◀──────────────────── B
   (একই সময়ে দুইদিকেই চলে)
   উদাহরণ: ফোন কল, আধুনিক Ethernet, TCP কানেকশন

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**Flutter-এ প্রাসঙ্গিকতা:**
- HTTP (REST API) → Half-Duplex (আপনি request পাঠান, server response দেয়)
- WebSocket → Full-Duplex (যেকোনো সময় যেকোনো দিক থেকে বার্তা আসতে পারে)

### ২.৮ Error Detection — ভুল ধরার পদ্ধতি

নেটওয়ার্কে ডেটা পাঠানোর সময় ভুল হতে পারে। এই ভুল ধরার তিনটি প্রধান পদ্ধতি:

```
① Parity Bit (সরলতম):
   ডেটা: 1010001 (7 bits)
   1-এর সংখ্যা: 3 (বিজোড়)
   Even parity bit যোগ: 10100011 → 1-এর সংখ্যা = 4 (জোড়)

   সমস্যা: দুটো ভুল ধরতে পারে না।

② Checksum:
   সব বাইট যোগ করো → যোগফলের complement = checksum
   UDP, IP প্রোটোকলে ব্যবহৃত।

③ CRC (Cyclic Redundancy Check):
   সবচেয়ে শক্তিশালী। Ethernet, Wi-Fi, TCP/IP-তে ব্যবহৃত।
   Polynomial division দিয়ে remainder বের করে।
```

## অধ্যায় ৩: নেটওয়ার্কের প্রকারভেদ — LAN থেকে ইন্টারনেট

> *"ইন্টারনেট আসলে লক্ষ লক্ষ ছোট নেটওয়ার্কের সমষ্টি।"*

### ৩.১ নেটওয়ার্ক কী?

দুই বা ততোধিক ডিভাইস একে অপরের সাথে যোগাযোগ করতে পারলে তাকে নেটওয়ার্ক বলে।

```
সবচেয়ে সরল নেটওয়ার্ক:

  ┌──────────┐                    ┌──────────┐
  │ Computer │────────────────────│  Server  │
  │    A     │    সরাসরি কেবল    │    B     │
  └──────────┘                    └──────────┘

এটি দিয়েও ফাইল শেয়ার করা যায়!
```

---

### ৩.২ নেটওয়ার্কের প্রকারভেদ — পরিধি অনুযায়ী

```
নেটওয়ার্কের পরিধির তুলনা:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  PAN (Personal Area Network)
  ├── পরিধি: ~10 মিটার
  ├── উদাহরণ: Bluetooth হেডফোন, স্মার্টওয়াচ, AirDrop
  └── ব্যবহার: ব্যক্তিগত ডিভাইস সংযোগ

  LAN (Local Area Network)
  ├── পরিধি: ~100 মিটার – 1 কিমি
  ├── উদাহরণ: অফিসের নেটওয়ার্ক, বাড়ির Wi-Fi, স্কুল নেটওয়ার্ক
  └── ব্যবহার: সীমিত এলাকায় সংস্থানের ভাগাভাগি

  MAN (Metropolitan Area Network)
  ├── পরিধি: ~5–50 কিমি (একটি শহর)
  ├── উদাহরণ: শহরের ক্যাবল TV নেটওয়ার্ক, বিশ্ববিদ্যালয় ক্যাম্পাস
  └── ব্যবহার: শহরের মধ্যে ISP সংযোগ

  WAN (Wide Area Network)
  ├── পরিধি: দেশ থেকে মহাদেশ
  ├── উদাহরণ: ইন্টারনেট নিজেই সবচেয়ে বড় WAN
  └── ব্যবহার: আন্তর্জাতিক যোগাযোগ

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**ভিজ্যুয়াল তুলনা:**

```
  আপনার ফোন ──(Bluetooth)──▶ হেডফোন           [PAN]
       │
  (Wi-Fi)
       │
  রাউটার ──── অফিসের সব কম্পিউটার             [LAN]
       │
  (ISP লাইন)
       │
  ঢাকার ISP নেটওয়ার্ক                          [MAN]
       │
  সমুদ্রের তলার Fiber Cable
       │
  যুক্তরাষ্ট্রের সার্ভার                         [WAN/Internet]
```

---

### ৩.৩ LAN — Local Area Network বিস্তারিত

LAN হলো সবচেয়ে পরিচিত নেটওয়ার্ক। আপনার অফিস বা বাসার নেটওয়ার্ক।

```
একটি ছোট অফিসের LAN:

                        ┌─────────────┐
                        │   Router    │◀──── ইন্টারনেট
                        │ (192.168.1.1)│
                        └──────┬──────┘
                               │
                        ┌──────┴──────┐
                        │   Switch    │
                        └─┬──┬──┬──┬─┘
                          │  │  │  │
                    ┌─────┘  │  │  └─────┐
                    │    ┌───┘  └───┐    │
                    ▼    ▼          ▼    ▼
                  [PC1][PC2]      [PC3][Printer]
               .1.100 .1.101    .1.102  .1.103

  সব ডিভাইস IP: 192.168.1.xxx (private)
```

**LAN-এর বৈশিষ্ট্য:**
- উচ্চ গতি: 100 Mbps – 10 Gbps
- কম latency: 1 ms এর কম
- নিজেদের মধ্যে সংযোগ সরাসরি
- সাধারণত একটি প্রতিষ্ঠানের নিয়ন্ত্রণে

---

### ৩.৪ WAN ও ইন্টারনেট — বিশ্বের বৃহত্তম নেটওয়ার্ক

**ইন্টারনেট** = Interconnected Networks — লক্ষ লক্ষ LAN ও MAN-এর সংযোগ।

```
ইন্টারনেটের স্তরবিন্যাস:

  ┌──────────────────────────────────────────────────────┐
  │                      Tier 1 ISPs                    │
  │   (AT&T, NTT, Deutsche Telekom — সমুদ্রের তলার     │
  │    ফাইবার ক্যাবল দিয়ে মহাদেশ সংযুক্ত করে)         │
  └──────────────────┬───────────────────────────────────┘
                     │  পিয়ারিং সংযোগ (বিনামূল্যে)
  ┌──────────────────┴───────────────────────────────────┐
  │                      Tier 2 ISPs                    │
  │   (আঞ্চলিক ISP — Tier 1 থেকে ব্যান্ডউইথ কেনে)      │
  └──────────────────┬───────────────────────────────────┘
                     │  ব্যান্ডউইথ বিক্রি করে
  ┌──────────────────┴───────────────────────────────────┐
  │                      Tier 3 ISPs                    │
  │   (BDIX, Grameenphone, Banglalink — সরাসরি ব্যবহারকারীদের│
  │    সংযোগ দেয়)                                       │
  └──────────────────┬───────────────────────────────────┘
                     │
          আপনার বাসার Router → আপনার Flutter App
```

**সমুদ্রের তলার ফাইবার ক্যাবল:**

```
বিশ্বের প্রধান Submarine Cable সংযোগ:
```

### ৩.৫ নেটওয়ার্ক Topology — সংযোগের আকৃতি

**Topology** মানে নেটওয়ার্কের ডিভাইসগুলো কিভাবে সংযুক্ত — এর গঠন বা আকৃতি।

#### ৩.৫.১ Bus Topology

```
Bus Topology:
  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
  │ PC-1 │  │ PC-2 │  │ PC-3 │  │ PC-4 │
  └──┬───┘  └──┬───┘  └──┬───┘  └──┬───┘
     │          │          │          │
━━━━━┷━━━━━━━━━┷━━━━━━━━━┷━━━━━━━━━┷━━━━━━  ← Backbone Cable
  Terminator                           Terminator

  ✅ সুবিধা: সহজ, সস্তা
  ❌ অসুবিধা: একটি ভাঙলে পুরো নেটওয়ার্ক বন্ধ
  📍 ব্যবহার: পুরনো Ethernet (Coaxial cable)
```

#### ৩.৫.২ Star Topology (সবচেয়ে প্রচলিত)

```
Star Topology:
         ┌──────┐
         │ PC-1 │
         └──┬───┘
            │
┌──────┐    │    ┌──────┐
│ PC-4 │────┤    ├──────│ PC-2 │
└──────┘    │    │      └──────┘
         ┌──┴────┴──┐
         │  Switch  │  ← কেন্দ্র
         └──┬───────┘
            │
         ┌──┴───┐
         │ PC-3 │
         └──────┘

  ✅ সুবিধা: একটি ভাঙলে বাকিরা চলে; সহজ troubleshoot
  ❌ অসুবিধা: Switch ভাঙলে সব বন্ধ (SPOF)
  📍 ব্যবহার: আধুনিক অফিস ও বাড়ির নেটওয়ার্ক
```

#### ৩.৫.৩ Ring Topology

```
Ring Topology:
       ┌──────┐
       │ PC-1 │
       └──┬───┘
    ╔══╝  ╚══╗
┌──────┐  ┌──────┐
│ PC-4 │  │ PC-2 │
└──────┘  └──────┘
    ╚══╗  ╔══╝
       └──┬───┘
       ┌──┴───┐
       │ PC-3 │
       └──────┘

  ✅ সুবিধা: প্রতিটি ডিভাইস সমান সুযোগ পায়
  ❌ অসুবিধা: একটি ভাঙলে পুরো নেটওয়ার্ক বিপদে
  📍 ব্যবহার: Token Ring (পুরনো), SONET/SDH (fiber)
```

#### ৩.৫.৪ Mesh Topology

```
Mesh Topology (আংশিক):
  ┌──────┐────────────┌──────┐
  │ PC-1 │            │ PC-2 │
  └──┬───┘            └──┬───┘
     │  ╲            ╱   │
     │    ╲  ╔════╗  ╱   │
     │      ╳║    ║╳      │
     │    ╱  ╚════╝  ╲   │
  ┌──┴───┐            └──┴───┐
  │ PC-4 │────────────│ PC-3 │
  └──────┘            └──────┘

  ✅ সুবিধা: অত্যন্ত নির্ভরযোগ্য; যেকোনো পথে যেতে পারে
  ❌ অসুবিধা: ব্যয়বহুল, জটিল
  📍 ব্যবহার: ইন্টারনেটের core routers, Military network
```

#### ৩.৫.৫ Hybrid Topology (বাস্তবে সবচেয়ে বেশি ব্যবহৃত)

```
আধুনিক কর্পোরেট নেটওয়ার্ক (Hybrid):

  ইন্টারনেট
      │
  ┌───┴────┐
  │ Router  │
  └───┬────┘
      │         ← WAN লিংক
  ┌───┴──────────────────────┐
  │         Core Switch       │    ← Star
  └───┬──────────┬────────────┘
      │          │
  ┌───┴──┐   ┌───┴──┐
  │Floor1│   │Floor2│    ← প্রতি ফ্লোরে Star
  │Switch│   │Switch│
  └┬─┬─┬┘   └┬─┬─┬─┘
   │ │ │      │ │ │
  PC PC PC   PC PC PC
```

### ৩.৬ WLAN — Wireless LAN ও Wi-Fi

Wi-Fi কিভাবে কাজ করে?

```
Wi-Fi আর্কিটেকচার:

  ইন্টারনেট
      │
  ┌───┴──────┐
  │  Router  │  ← Gateway + DHCP Server + NAT
  │(Wireless)│
  └────┬─────┘
       │ 2.4GHz / 5GHz Radio Waves
    ╔══╧══╗
    ║ Air ║   ← মাধ্যম (তার ছাড়া!)
    ╚══╤══╝
  ┌────┴────┐    ┌─────────┐    ┌─────────┐
  │  Phone  │    │  Laptop │    │  TV     │
  └─────────┘    └─────────┘    └─────────┘

Wi-Fi Standards:
  802.11b  → 11 Mbps  (2.4GHz)  — পুরনো
  802.11g  → 54 Mbps  (2.4GHz)
  802.11n  → 600 Mbps (2.4/5GHz) — Wi-Fi 4
  802.11ac → 3.5 Gbps (5GHz)    — Wi-Fi 5
  802.11ax → 9.6 Gbps (2.4/5/6GHz) — Wi-Fi 6
```

**2.4 GHz vs 5 GHz:**

| বৈশিষ্ট্য | 2.4 GHz | 5 GHz |
|-----------|---------|-------|
| পরিধি | বেশি (দেওয়াল ভেদ করে) | কম |
| গতি | কম | বেশি |
| ভিড় | বেশি (সব ডিভাইস) | কম |
| উপযুক্ত | বড় বাড়ি, দেওয়ালের পাশে | কাছাকাছি, হাই-স্পিড দরকার |
